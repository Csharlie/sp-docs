# CMS Runtime Parity Gate

## Summary (Simple)

Platform-szintű szabály: az adapter runtime switch (JSON → WP) production cutover **tilos**, amíg a WP tartalom nem egyezik a site.ts tartalommal. A `verify-parity.ts` PASS eredménye előfeltétel. Ez védi a rendszert attól, hogy a WP-re való átállás után a frontend hibás vagy hiányos adatokat kapjon.

## Technical Explanation

A Spektra platform kétféle adatforrást támogat:
1. **JSON mód**: a frontend közvetlenül a `site.ts`-ből (statikus import) olvas
2. **WP mód**: a frontend a `/spektra/v1/site` REST endpoint-ból olvas (WordPress ACF adatok)

Az adapter switch az a döntés, amellyel a frontend az egyik forrásról a másikra vált. Ha a WP-ben nem ugyanaz az adat van, mint a site.ts-ben, a switch után a frontend törött vagy hiányos tartalmat jelenít meg.

### Gate feltételek

A production cutover előtt az alábbi **összes** feltételnek teljesülnie kell:

1. **Pipeline PASS**: `seed-pipeline.ps1` sikeresen lefutott
2. **55/55 field match**: `verify-parity.ts` 0 FAIL, 0 missing
3. **18/18 endpoint smoke**: `verify-endpoint.php` minden image field valid URL
4. **0 missing field**: minden seed.json kulcs megjelenik wp-state.json-ben
5. **Media library sync**: minden image a WP media library-ben van (sideloaded)

### Mikor kell újrafuttatni

- **Minden site.ts tartalomváltozás után** — regresszió védelem
- **Minden ACF regisztráció módosítás után** — mező elérhetőség változhat
- **Adapter switch production cutover előtt** — gate feltétel

## Why It Exists

Phase 8.3-ban bevezettük az adapter switch-et, de nem volt biztosíték, hogy a WP adatbázisban lévő tartalom megegyezik a site.ts-sel. A dummy adatok (Phase 6.4-ből) és a valós tartalom között nem volt ellenőrzési mechanizmus. A Parity Gate megakadályozza, hogy a switch előtt ne legyen igazolt egyezés.

## Implementation Notes

- **verify-parity.ts** (sp-infra/seed/): összehasonlít seed.json vs wp-state.json
  - Text/CTA: exact match
  - Image: structural match (URL eltérhet, alt egyeznie kell)
  - Exit 0 = PASS, 1 = FAIL
- **verify-endpoint.php** (sp-infra/seed/): belső REST dispatch, image URL validáció
- **seed-pipeline.ps1** (sp-infra/seed/): orchestrálja a teljes flow-t és összesíti az eredményt

A gate jelenleg **manuális** — a pipeline PASS/FAIL eredményt a fejlesztő ellenőrzi. CI-integrált automatikus gate a P11+ scope.

## Boundaries and Guardrails

- **Owner**: platform réteg (sp-infra)
- **Érvényesség**: minden kliens repóra vonatkozik, nem csak sp-benettcar-ra
- **NEM szabad**: a gate-et megkerülni (`--no-verify` flag nincs és nem is lesz)
- **NEM szabad**: a parity definíciót kliens-specifikusan lazítani — a szabály platform-szintű
- **Navigation kívül esik**: config-driven, nem ACF, nem a parity scope része
