# Runtime Cutover Baseline – P9.1 Gate

## Summary (Simple)

A P9.1 gate 5 ellenőrzési pontból álló egységes belépési feltétel, amely megerősíti, hogy a frontend biztonságosan átválthat JSON módból WordPress módra. Ez egy külön gate P9.2-től, amely a vizuális E2E verifikáció (bc-hero section) — a P9.1 csak a runtime infrastruktúra épségét igazolja, adattartalomtól függetlenül.

## Technical Explanation

A gate 5 checkpoint-ból áll, amelyeket sorban kell teljesíteni:

### CP1 — .env switch

Az `sp-clients/sp-benettcar/.env` fájlban:

```
VITE_DATA_SOURCE=wordpress
VITE_WORDPRESS_API_BASE=http://benettcar.local
```

Ha `VITE_DATA_SOURCE` ismeretlen értékre van állítva, a `createAdapter()` factory JSON módra esik vissza és `console.warn`-t küld. Az env értéket csak modul inicializálásakor olvassa — nem requestenként.

### CP2 — Endpoint health

```powershell
irm "http://benettcar.local/wp-json/spektra/v1/site"
```

Elvárt válasz:
- HTTP 200 OK
- Top-level kulcsok: `site`, `navigation`, `sections`
- `sections` tömb: legalább annyi elem, amennyi a seeded tartalomban van (P9.1 esetén 10)

### CP3 — CORS verifikáció

```powershell
Invoke-WebRequest -Uri "http://benettcar.local/wp-json/spektra/v1/site" \
  -Headers @{ Origin = "http://localhost:5174" }
```

Elvárt response header-ek:
- `Access-Control-Allow-Origin: http://localhost:5174`
- `Access-Control-Allow-Credentials: true`
- `Access-Control-Allow-Methods: OPTIONS, GET, POST, PUT, PATCH, DELETE`

CORS konfiguráció helye:
- Engedélyezett origin-ek: `sp-clients/sp-benettcar/infra/config.php` → `allowed_origins`
- Végrehajtás: `sp-infra/plugin/spektra-api/includes/class-cors.php`

### CP4 — No-silent-fallback verifikáció (kódelemzés)

Meggyőzés arról, hogy hiba esetén a rendszer **sosem csap vissza JSON adatokra**.

- `sp-platform/packages/data/src/wordpress.ts`: HTTP non-2xx → `throw new Error('WordPress API error: {status} {statusText}')`
- `sp-platform/packages/runtime/src/context.tsx`: catch → `setState({ data: null, loading: false, error })` — az adat `null`, nem fallback JSON
- `VITE_DATA_SOURCE` nincs runtime-ban ellenőrizve — csak adapter létrehozáskor

Részletek: lásd [p9.1-no-silent-fallback.md](p9.1-no-silent-fallback.md)

### CP5 — Error UI meglét (kód változás)

Az `sp-clients/sp-benettcar/src/App.tsx`-ben az `error` prop hiánya a `LandingTemplate`-en azt okozta, hogy WP hiba esetén a template belső default `<div role="alert">` doboz jelent meg — branding nélkül.

Fix: `error` prop implementálása branded, Magyar szövegű error UI-jal.

Részletek: lásd [p9.1-error-ui-pattern.md](p9.1-error-ui-pattern.md)

### Gate státusz

A gate akkor PASS, ha mind az 5 checkpoint teljesül. Csak akkor szabad P9.2-re lépni (bc-hero E2E vizuális verifikáció), ha a P9.1 gate PASS.

## Why It Exists

A P8.5 Parity Gate a tartalom helyességét ellenőrzi (WP adatok == site.ts adatok). A P9.1 gate külön felelősségi körre fókuszál: az infrastruktúra és a runtime saját épségére — env beállítás, hálózati elérhetőség, CORS, hibaviselkedés. Ez azért különálló gate, mert megbukhat akkor is, ha a tartalom helyes — pl. rossz CORS config vagy hiányzó error UI esetén.

P9.2 szándékosan nem foglalkozik ezekkel az infra feltételekkel — az csak visual E2E.

## Implementation Notes

- `.env` fájl: `sp-clients/sp-benettcar/.env`
- Adapter factory: `sp-clients/sp-benettcar/src/data/create-adapter.ts`
- WP adapter: `sp-platform/packages/data/src/wordpress.ts`
- Runtime context: `sp-platform/packages/runtime/src/context.tsx`
- Landing template error slot: `sp-platform/packages/templates/src/LandingTemplate.tsx`
- CORS plugin: `sp-infra/plugin/spektra-api/includes/class-cors.php`
- CORS konfig: `sp-clients/sp-benettcar/infra/config.php`
- App entry: `sp-clients/sp-benettcar/src/App.tsx`

## Boundaries and Guardrails

- **Owner**: platform + infra + client réteg közös felelősség — minden checkpoint más réteget érint
- **NEM szabad**: P9.1 FAIL állapotban P9.2-re lépni
- **NEM szabad**: az 5 checkpoint-ot részlegesen elfogadottnak tekinteni — mind az 5 kötelező
- **NEM szabad**: a CORS-t a platformban hardkódolni — az mindig infra+client config felelősség
- A gate manuális folyamat — CI automatizálás jövőbeli scope (P11+)
