# Error UI Pattern – LandingTemplate error prop implementáció

## Summary (Simple)

A `LandingTemplate` platformszintű template egy `error` propot biztosít, amelyet a kliensnek kell kitölteni branded error UI-jal. Ha a prop hiányzik, a template egy branding nélküli default `<div role="alert">` dobozt jelenít meg hiba esetén. A `sp-benettcar` kliens P9.1-ben implementálta ezt a propot magyarul szövegezett, branded error UI-jal.

## Technical Explanation

### A template error slot

**Fájl**: `sp-platform/packages/templates/src/LandingTemplate.tsx`

A `LandingTemplate` az `error` propot `(err: Error) => ReactNode` típusú render function-ként fogadja. Ha hiba van (WP API fail), a template ezt a render function-t hívja az error objektummal.

Ha a kliens nem adja meg a propot, a template belső default-ja lép életbe:

```tsx
// platform belső default — branding nélküli
<div role="alert">{error.message}</div>
```

Ez elfogadható accessibility szempontból, de:
- Nincs stílus, nincs branding
- A `error.message` technikai szöveg — pl. `"WordPress API error: 503 Service Unavailable"`
- A felhasználó számára érthetetlen, ijesztő

### A P9.1 előtti állapot (`sp-benettcar`)

**Fájl**: `sp-clients/sp-benettcar/src/App.tsx`

Az `error` prop **hiányzott** a `LandingTemplate` JSX-ből. Hiba esetén a template platforma default-ja aktivált — technikai szöveg, branding nélkül.

### A P9.1-ben implementált branded error UI

**Fájl**: `sp-clients/sp-benettcar/src/App.tsx`

```tsx
<LandingTemplate
  adapter={adapter}
  sections={sectionRegistry}
  error={(error) => (
    <div className="min-h-screen flex items-center justify-center bg-background">
      <div className="text-center">
        <p className="text-destructive font-semibold mb-2">A tartalom nem elérhető</p>
        <p className="text-sm text-muted-foreground">{error.message}</p>
      </div>
    </div>
  )}
/>
```

Jellemzők:
- Teljes viewport-ot lefedi (`min-h-screen`)
- Tailwind utility classes — a kliens design system token-jei (`bg-background`, `text-destructive`, `text-muted-foreground`)
- Magyar szöveg — „A tartalom nem elérhető"
- A technikai `error.message` megjelenik (`text-sm`), de vizuálisan másodlagos
- Platform template `error` prop → `(err: Error) => ReactNode` — a kliens teljesen kontrollálja a megjelenést

### Miért render function és nem ReactNode

A `LandingTemplate` render function-t vár (`(err: Error) => ReactNode`), nem statikus `ReactNode`-ot. Ez azért, mert a kliens hozzáférést kap az `Error` objektumhoz, és annak `message` tartalmát megjelenítheti — vagy más döntést hozhat az error típusa alapján.

## Why It Exists

A platform template nem tartalmazhat kliens-specifikus szöveget, brandingot vagy dizájnt. Az error UI felelőssége ezért a kliensnél van. A `LandingTemplate` csak a slot infrastruktúráját biztosítja.

P9.1-ben ez a felelősséghatár explicit lett, mert a CP5 checkpoint megbukott volna a branded error UI nélkül — a Runtime Cutover Baseline elvárás, hogy a kliens hiba esetén érthető, branded UI-t mutasson.

## Implementation Notes

- **Template error slot**: `sp-platform/packages/templates/src/LandingTemplate.tsx`
  - Prop neve: `error`
  - Típus: `(err: Error) => ReactNode`
  - Ha nincs megadva: `<div role="alert">{err.message}</div>` platform default
- **Kliens implementáció**: `sp-clients/sp-benettcar/src/App.tsx`
  - `error` prop hozzáadva a `LandingTemplate`-hez
  - Tailwind: `min-h-screen`, `flex`, `items-center`, `justify-center`, `bg-background`, `text-destructive`, `text-muted-foreground`
  - Magyar szöveg: „A tartalom nem elérhető"
  - `{error.message}` megjelenítve másodlagos stílussal
- **Error origin**: a hiba a `sp-platform/packages/runtime/src/context.tsx`-ből érkezik — `{ data: null, error }` state

## Boundaries and Guardrails

- **Owner**: kliens réteg (branded megjelenés) + platform réteg (slot infrastruktúra)
- **NEM szabad**: a platform template-ben kliens-specifikus szöveget/stílust hardkódolni
- **NEM szabad**: az `error` propot elhagyni — a platform default nem elégséges production UI-nak
- **NEM szabad**: a platform default-ot változtatni kliens-specifikus okokból
- **Minden kliens repóban** önállóan implementálandó az `error` prop — nincs megosztott error UI komponens platform szinten (szándékos: különböző kliens brandingok)
- A branded szöveg (`A tartalom nem elérhető`) kliens-specifikus — más kliensek más szöveget használhatnak
